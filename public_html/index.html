shebang.js

... and the information disclosure is gone reduced

It is well known that there are a number of issues with passing parameters within a query string:

  • the URI (including query string) is displayed in the address bar (and could be observed by a "shoulder surfer")
  • if the page is printed, the URI may be printed too
  • URLs are frequently logged on the server (and possibly on any proxy servers in-between)
  • the URI can be bookmarked (and potentially synced using an untrusted internet service)
  • the URI will be written to the history (which is usually written to disk)

All of these issues (except the final issue) can be remediated using shebang.js - a tool to pass parameters as part of the fragment of a URI, where it is not feasible to use an alternative method (e.g. POST).

Instead of...

https://www.example.com/login?secret=ztpHck7...

You should...

https://www.example.com/login#!secret=ztpHck7...
